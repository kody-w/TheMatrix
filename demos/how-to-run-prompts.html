üî¨ How The Matrix Orchestration Works

The Orchestration Pattern

The Matrix uses a hierarchical agent architecture:

Orchestrator (200k context - Claude in your session)
‚îú‚îÄ‚îÄ Discovery & Analysis (reads project context)
‚îú‚îÄ‚îÄ Strategy Generation (creates work breakdown structure)
‚îú‚îÄ‚îÄ System Analysis (extracts patterns, standards, schemas)
‚îî‚îÄ‚îÄ Parallel Agent Spawning
    ‚îú‚îÄ‚îÄ N √ó outcome-generator agents (parallel execution)
    ‚îî‚îÄ‚îÄ 1 √ó integrator agent (synthesizes results)

The 6-Step Workflow

  1. Discovery & Analysis: Claude scans your project files to understand context
  2. Work Breakdown: Creates N packages with M items each (N√óM total outcomes)
  3. Agent Briefing: Prepares comprehensive instructions for all agents
  4. Parallel Spawning: Launches N agents simultaneously (NOT sequential)
  5. Integration: Integrator agent synthesizes all N√óM outcomes
  6. Reporting: Complete documentation of what was built
üí° Key Insight: The orchestrator preserves its 200k context by delegating ALL implementation to subagents. This enables coordinating hundreds of parallel agents.

üîß Troubleshooting

Common Issues

Issue: "Claude doesn't recognize The Matrix framework"
Solution: Make sure you're in The Matrix directory and .claude/CLAUDE.md exists. Add "Use The Matrix orchestration framework" to your prompt.
Issue: "Agents running sequentially instead of parallel"
Solution: Check that your prompt explicitly mentions scale (e.g., "200 components"). The orchestrator needs clear scale to spawn multiple agents.
Issue: "Generated code has errors"
Solution: Include "with comprehensive testing" in your prompt. Test suites catch issues during generation.
